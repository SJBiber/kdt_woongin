# ⏰ Cron 서비스를 이용한 로컬 프로세스 자동화 (Mac/Linux)

컴퓨터를 종료하지 않는 환경에서 특정 시간에 스크립트를 주기적으로 실행하고 싶으신가요? 유닉스 계열 운영체제에서 제공하는 표준 시간 기반 작업 스케줄러인 **Cron**을 활용하는 방법을 안내합니다.

---

## 💡 주요 개념 (Terminology)
- **Cron**: 백그라운드에서 주기적으로 특정 작업을 실행해주는 시스템 서비스입니다.
- **Crontab**: Cron 테이블의 줄임말로, 실행할 작업 목록을 담고 있는 설정 파일입니다.
- **절대 경로(Absolute Path)**: 파일의 위치를 `/Users/name/project/example.py`와 같이 전체 주소로 적는 방식입니다. (로봇이 파일을 헤매지 않고 찾게 해줍니다.)

---

## 🚀 설정 가이드 (CLI 기반)

### 1단계: Crontab 편집 모드 진입
시스템 터미널(Terminal)을 실행한 뒤 아래 명령어를 입력하여 편집기를 엽니다.
```bash
crontab -e
```
*(`-e`는 Edit의 약자입니다. 처음 실행 시 편집기 번호를 물어보면 보통 `1`번(nano)을 선택합니다.)*

### 2단계: 크론 표현식(Cron Expression) 작성
파일 최하단에 아래와 같이 실행 규칙을 입력합니다.
```bash
# 매일 오전 9시 0분에 Python3를 사용하여 지정된 스크립트를 실행합니다.
0 9 * * * /usr/bin/python3 /절대/경로/example.py
```

**[표현식 해석]**
- `0 9 * * *`: 분(0~59) 시(0~23) 일(1~31) 월(1~12) 요일(0~6) 순서입니다.
- `/usr/bin/python3`: 파이썬 실행 파일의 경로입니다.
- `/배승재/집갈래/example.py`: 실행할 타겟 파일의 **절대 경로**입니다.

### 3단계: 저장 및 편집기 종료
- **nano 편집기**: `Ctrl + O` (작성), `Enter` (저장), `Ctrl + X` (종료) 순으로 진행합니다.
- **vim 편집기**: `Esc` 키를 누른 뒤 `:wq` (Write & Quit) 입력 후 엔터를 칩니다.

---

## ✅ 설정 확인 (Validation)
정상적으로 등록되었는지 확인하려면 아래 명령어를 사용합니다.
```bash
crontab -l
```
*(`-l`은 List의 약자이며, 현재 등록된 모든 자동화 목록을 출력합니다.)*

---

## 💡 운영 시 주의사항
1. **서버 가동 상태**: 로컬 컴퓨터가 '잠자기' 모드이거나 전원이 꺼진 상태에서는 작업이 수행되지 않습니다.
2. **로그 확인**: 작업이 정상적으로 수행되었는지 확인하려면 출력 결과를 로그 파일로 리다이렉트(`> logs/cron.log 2>&1`)하는 설정을 권장합니다.
