# 🗓️ Windows 작업 스케줄러를 활용한 로컬 자동화 가이드

윈도우 운영체제 환경에서 특정 시간마다 파이썬 스크립트를 실행하고 싶으신가요? 리눅스의 Cron과 대응되는 윈도우 표준 도구인 **작업 스케줄러(Task Scheduler)** 설정 방법을 안내합니다.

---

## 💡 주요 개념 (Terminology)
- **작업 스케줄러(Task Scheduler)**: 윈도우에서 특정 조건(시간, 이벤트 등)이 만족될 때 프로그램을 실행해주는 내장 관리 도구입니다.
- **트리거(Trigger)**: 작업이 시작되는 '방당쇠' 역할을 하는 조건입니다. (예: 매일 오전 9시)
- **동작(Action)**: 트리거가 발동했을 때 컴퓨터가 수행할 구체적인 행동입니다. (예: 파이썬 실행)
- **인수(Arguments)**: 실행 프로그램 뒤에 붙는 추가 옵션으로, 주로 실행할 스크립트 파일의 경로를 기입합니다.

---

## 🚀 설정 가이드 (GUI 기반)

### 1단계: 작업 스케줄러 실행
1. 윈도우 검색창에 **'작업 스케줄러'**를 입력하여 실행합니다.
2. 오른쪽 [작업] 패널에서 **[기본 작업 만들기...]**를 클릭합니다.

### 2단계: 트리거(시간) 설정
1. **이름**: 본인이 알아보기 쉬운 이름(예: `Naver_News_Collector`)을 입력합니다.
2. **작업 트리거**: 실행 주기를 선택합니다. (보통 '매일' 권장)
3. **매일**: 시작 시간과 반복 주기를 설정합니다. (예: 오전 9:00:00)

### 3단계: 동작(프로그램 실행) 설정
이 부분이 가장 중요합니다.

1. **동작**: '프로그램 시작'을 선택합니다.
2. **프로그램/스크립트**: 본인의 파이썬 실행 파일(`python.exe`) 경로를 입력합니다.
   - 찾기 어려우면 터미널(CMD)에서 `where python`을 입력하여 나오는 경로를 복사하세요.
3. **인수 추가(옵션)**: 실행할 파이썬 파일의 **절대 경로**를 입력합니다.
   - 예: `C:\Users\Name\Project\example.py`
4. **시작 위치(옵션)**: 스크립트가 들어있는 폴더 경로를 입력합니다. (파일 경로에서 파일명만 뺀 주소)

---

## ✅ 정상 작동 테스트
1. 작업 스케줄러 라이브러리 목록에서 내가 만든 작업을 찾습니다.
2. 마우스 오른쪽 버튼을 눌러 **[실행]**을 클릭합니다.
3. 잠시 후 결과 파일이 생성되거나 데이터가 DB에 들어왔는지 확인합니다.

---

## 💡 윈도우 환경 주의사항
1. **절전 모드**: 윈도우 설정에서 컴퓨터가 '절전' 상태로 들어가지 않도록 하거나, 작업 설정의 [조건] 탭에서 **'이 작업을 실행하기 위해 컴퓨터의 대기 모드를 종료'** 옵션을 체크해야 합니다.
2. **백그라운드 실행**: 스크립트 실행 시 검은색 윈도우 창이 뜨는 게 싫다면, 일반 탭에서 **'사용자가 로그온되어 있는지 발송 여부에 관계없이 실행'**을 선택하세요.
