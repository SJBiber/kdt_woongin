# 🤖 GitHub Actions 상세 아키텍처 및 설정 가이드

본 문서는 GitHub Actions를 활용한 데이터 파이프라인 구축 시 필요한 단계별 인터페이스 조작과 환경 구성 방법을 상세히 다룹니다.

---

## 1. 개요 및 장점
GitHub Actions는 코드 체크아웃, 환경 구축, 스크립트 실행에 이르는 일련의 작업을 자동화합니다. 클라우드 기반으로 운영되므로 별도의 서버 비용 없이 안정적으로 리소스를 확보할 수 있습니다.

---

## 2. 세부 설정 단계 (Step-by-Step)

### Phase 1: 저장소 보안 환경 구축 (Secrets)
보안상 안전한 환경 변수 주입 시스템을 구축합니다.

1. **저장소 상단 배너**의 **[Settings]**를 클릭합니다. (저장소 관리 권한 필요)
2. **Security 섹션** 내 **[Secrets and variables]** 하위의 **[Actions]** 페이지로 진입합니다.
3. **[New repository secret]**을 클릭하여 아래 예시와 같이 환경 변수를 매핑합니다.
   - **Name**: `SUPABASE_URL` (대문자 및 언더바 사용 권장)
   - **Secret**: 실제 데이터베이스 접근 URL
4. **[Add secret]**을 완료하여 저장소 수준의 환경 변수 등록을 마칩니다.

### Phase 2: 워크플로우 실행 및 모니터링 (Monitoring)
정의된 파이프라인이 정상적으로 작동하는지 확인하는 과정입니다.

1. 상부 메뉴 바의 **[Actions]** 탭을 클릭하여 대시보드로 이동합니다.
2. 좌측 패널의 **Workflows** 목록에서 실행 대상 항목(예: `일일 자동화 로봇`)을 선택합니다.
3. 실시간 테스트가 필요한 경우 **[Run workflow]** 드롭다운 버튼을 눌러 수동 트리거(Trigger)를 발생시킵니다.
4. 실행 결과(Run) 목록에서 상태 아이콘을 확인합니다:
   - 🟡 **In-progress**: 현재 Runner에서 작업을 수행 중입니다.
   - ✅ **Success**: 선언된 모든 Step이 정상 종료되었습니다.
   - ❌ **Failure**: 작업 수행 중 에러(Exit Code 1 등)가 발생했습니다. 로그를 열어 구문을 분석해야 합니다.

---

## 3. 운영 FAQ

### Q. 작업 실행 시간이 초과되는 경우
GitHub 아카이브 및 무료 환경에서의 Job 수행 시간은 최대 6시간입니다. 대량의 데이터 처리 시 스크립트를 분할 실행하는 전략이 필요합니다.

### Q. 실패 알림 수신 설정
저장소의 **[Settings]** -> **[Notifications]**에서 작업 실패 시 특정 이메일로 리포트를 발송하도록 설정할 수 있습니다.

---

## 💡 주요 구문 해석 (YAML Snippet)
- `cron: '0 0 * * *'`: 국제 표준시(UTC) 기준 00:00입니다. 한국 시간(KST)은 9시간 빠른 **09:00**에 해당합니다.
- `uses: actions/checkout@v3`: GitHub 저장소의 소스 코드를 가상 머신(Runner)의 작업 디렉토리로 복제해오는 과정입니다.
