# 🛠️ GitHub Actions 자동화 운영 트러블슈팅 (FAQ)

자동화 워크플로우 운영 중 발생할 수 있는 주요 예외 상황과 해결 방법을 안내합니다. 시스템이 의도한 대로 작동하지 않을 때 아래 체크리스트를 기반으로 디버깅을 수행하세요.

---

## 1. 워크플로우 목록 미노출 현상 (Workflow Parsing Failure)
파일을 업로드했음에도 [Actions] 탭에서 확인되지 않는 경우입니다.

- **원격 저장소 반영 여부**: 로컬에서 생성한 파일이 `git push`를 통해 GitHub 서버로 최종 업로드되었는지 확인하세요.
- **예약어 오타 및 경로 확인**:
  - 폴더명은 반드시 `.github/workflows/` (마침표 및 복수형 s 포함)여야 합니다.
  - 파일 확장자는 `.yml` 또는 `.yaml` 중 하나여야 합니다.
- **YAML 구문 오류**: 들여쓰기(Indentation) 오류가 있을 경우 GitHub가 파일을 워크플로우로 인식하지 못합니다. (AI 가이드를 활용해 구문을 검증하세요.)

---

## 2. 파일명 및 명명 규칙 (Naming Conventions)
- **자유도**: 파일명 자체는 작업 성격에 맞게 자유롭게 지정할 수 있습니다 (예: `daily_collector.yml`).
- **영향**: 파일명은 대시보드 URL과 로그 구분에 사용되므로 영문 소문자와 언더바(`_`) 또는 대시(`-`) 조합을 권장합니다.

---

## 3. 원격지 서버 인코딩 및 숨김 파일 (OS Characteristics)
- **숨김 속성**: `.github` 폴더와 같이 마침표로 시작하는 이름은 유닉스 계열(Linux/Mac)에서 시스템 숨김 파일로 처리됩니다. 터미널에서 `ls -a` 명령어로 존재 여부를 확인하거나, GitHub 웹 인터페이스에서 직접 관리하는 것이 안전합니다.

---

## 4. 실행 결과 실패 (Job Failure Analysis)
워크플로우가 빨간색 아이콘(Failure)으로 종료된 경우입니다.

- **상세 로그 분석**: 실패한 Run을 클릭하고 각 Step의 상세 로그를 펼쳐보세요. 오류 메시지 상단에 명시된 행 번호(Line Number)가 핵심 단서입니다.
- **주요 원인**: 
  - **Secrets 누락**: `env` 섹션에서 요구하는 보안 변수가 GitHub Settings에 등록되지 않은 경우
  - **모듈 임포트 에러**: 가상 서버(Runner)에 `requirements.txt`에 정의된 의존 패키지가 정상적으로 설치되지 않은 경우

---

## 💡 효율적인 디버깅 팁
1. **AI 적극 활용**: 발생한 에러 로그 전체를 복사하여 AI(ChatGPT 등)에게 넘겨주세요. "현재 내 YAML 설정에서 이 에러가 발생하는 원인을 찾아줘"라고 요청하면 즉각적인 수정을 받을 수 있습니다.
2. **리트라이(Retry)**: 일시적인 네트워크 이슈일 수 있으므로 **[Re-run failed jobs]**를 통해 재작업을 지시해보세요.

---

**지속적인 모니터링과 빠른 피드백 반영은 자동화 시스템의 안정성을 결정짓습니다.**
