# ☁️ GitHub Actions를 활용한 클라우드 자동화 가이드

개인용 워크스테이션을 24시간 가동할 수 없는 환경이라면 파이프라인 자동화 도구인 **GitHub Actions**를 활용하여 서버리스 환경에서 스크립트를 주기적으로 실행할 수 있습니다.

---

## 💡 주요 개념 (Terminology)
- **Workflow**: GitHub 저장소에서 실행되는 자동화된 전체 프로세스입니다.
- **Job**: 하나의 Workflow 내에서 실행되는 독립적인 작업 단위입니다.
- **Runner**: GitHub에서 제공하는 가상 서버(Ubuntu, Windows 등)로, 코드가 실제로 실행되는 환경입니다.
- **Secrets**: API 키나 DB 패스워드 등 외부에 노출되어서는 안 되는 보안 데이터를 안전하게 관리하는 환경 변수입니다.

---

## 🚀 워크플로우 운영 프로세스

### 1단계: 보안 환경 변수(Secrets) 관리
로봇이 작업을 수행하기 위해 필요한 인증키들을 환경 변수로 등록합니다.

1. 해당 GitHub 저장소(Repository)의 **[Settings]** 탭으로 이동합니다.
2. 왼쪽 사이드바에서 **[Secrets and variables]** -> **[Actions]**를 선택합니다.
3. **[New repository secret]**을 클릭하여 아래 항목들을 등록합니다.
   - 예: `YOUTUBE_API_KEY`, `SUPABASE_KEY` 등
   - 등록된 정보는 마스킹 처리되어 팀원들에게도 노출되지 않습니다.

### 2단계: 워크플로우 정의 파일(YAML) 업로드
저장소 내 경로 `.github/workflows/*.yml`에 작업 정의서를 생성합니다. 이 파일에는 실행 주기(Cron), OS 환경, 실행 명령어 등이 명시됩니다.

### 3단계: 수동 및 자동 트리거링(Triggering)
- **자동**: 설정된 Cron 주기에 따라 시스템이 자동 실행합니다.
- **수동**: **[Actions]** 탭에서 특정 워크플로우를 선택한 뒤 **[Run workflow]** 버튼을 통해 즉시 실행할 수 있습니다.
