# 📋 로봇 "업무 지시서" (YAML) 쉽게 쓰는 법

로봇에게 일을 시키려면 **"업무 지시서"**가 필요합니다. 이 지시서는 컴퓨터가 읽기 편하도록 `.yml` (야몰)이라는 형식을 사용합니다. 

어려워 보이지만, 딱 세 가지만 기억하면 누구나 쓸 수 있습니다!

---

## 💡 지시서의 위치 (이게 제일 중요해요! ⭐)

로봇은 정해진 위치에 있는 지시서만 읽습니다. 아래 폴더 구조를 꼭 확인하세요!

```text
[내 저장소]
  └── .github (폴더)
        └── workflows (폴더)
              └── 어떤이름든상관없음.yml (지시서 파일)
```
*주의: 폴더 이름 앞에 마침표(`.`)가 있거나, `s`가 붙는 것을 꼭 확인하세요!*

## 💡 지시서의 3요소 (언제, 어디서, 무엇을)

### 2. 어디서 실행할까? (`runs-on:`)
로봇이 일할 장소(서버)를 빌리는 것입니다.
- 보통 `ubuntu-latest`라고 적으면 "가장 튼튼하고 빠른 컴퓨터를 빌려줘!"라는 뜻이 됩니다.

### 3. 무엇을 실행할까? (`steps:`)
로봇이 순서대로 해야 할 행동을 하나씩 적어줍니다.
- **체크아웃**: "내 가방(코드/파일)을 챙겨와!"
- **도구 설치**: "파이썬(Python) 도구를 설치해!"
- **실행**: "이제 준비됐으니 `python example.py`를 실행해!"

---

## 🚀 바로 쓸 수 있는 템플릿 (복사해서 사용하세요!)

아래 코드를 복사해서 내 저장소의 `.github/workflows/` 폴더 안에 아무 이름(예: `auto.yml`)으로 저장하면 바로 작동합니다.

```yaml
name: 자동화 로봇 지시서

on:
  schedule:
    - cron: '0 0 * * *'        # 매일 아침 9시 출근
  workflow_dispatch:           # 수동 버튼 활성화

jobs:
  my_work:
    runs-on: ubuntu-latest    # 가상 컴퓨터 빌리기
    steps:
      - uses: actions/checkout@v3     # 파일 챙기기
      - uses: actions/setup-python@v4 # 파이썬 설치
        with:
          python-version: '3.11'
          
      - name: 실행하기
        run: python example.py        # 내 파일 이름 적기!
```

## ⚠️ 작성 시 주의할 점 (실수하기 쉬운 곳)

1. **들여쓰기(칸 띄우기)가 생명!**
   - YAML 파일은 들여쓰기로 순서를 구분합니다. 삐뚤빼뚤하면 로봇이 지시서를 읽지 못해요. (보통 공백 2칸씩 사용합니다.)
   
2. **이름 바꾸기**
   - 맨 아래 `python example.py` 부분에 본인이 실제로 만든 파일 이름을 적어줘야 합니다. 만약 파일이 `start.py`라면 이 부분을 바꿔주세요.

3. **비밀번호가 필요할 때**
   - API 키 같은 중요한 비밀번호는 직접 적지 말고, 우리가 배운 **"보안 금고(Secrets)"**에 넣은 뒤 `${{ secrets.이름 }}` 형식으로 로봇에게 전달해주세요.
