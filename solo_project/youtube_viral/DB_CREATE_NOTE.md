# 📝 Database Design Note (v2.5 Snapshot Edition)

## 📌 설계 변경 배경: "전수 추적"에서 "스냅샷 분석"으로
기존의 `video_series_data` 방식은 개별 영상의 지표를 실시간으로 추적하기 위해 반복적인 API 호출과 방대한 로그 데이터 적재가 필요했습니다. 이는 YouTube API 할당량을 빠르게 소진시키고 DB 조회 성능을 저하시키는 원인이 되었습니다.

v2.5에서는 **Snapshot Analysis** 기법을 도입하여, 특정 시점의 데이터 셋 전체를 통계적으로 분석하고 결과값만 저장하는 방식을 채택했습니다.

## 🚀 주요 개선 사항

### 1. 전용 분석 테이블 (`trend_analysis`)
*   **Topic Share**: 인기 급상승 영상 300개를 하나의 표본 집단으로 보고, 특정 주제의 점유율을 즉시 산출합니다.
*   **Power Score (Velocity)**: 영상을 매번 추적하지 않고, `현재 조회수 / 업로드 후 경과 시간` 공식을 통해 실시간 성장세를 바로 포착합니다.
*   **Recent Ratio**: 검색 결과 50개 중 최근 24시간 내 업로드된 영상의 비율을 통해 트렌드의 '신선도'를 측정합니다.
*   **Avg Views & Total Vols**: 키워드별 평균 조회수와 총 수집된 영상의 볼륨을 기록하여, 해당 트렌드의 규모(Scale)를 파악할 수 있게 합니다.

### 2. Quota & Storage Efficiency
*   개별 영상의 히스토리를 쌓지 않으므로 데이터 쓰기량이 90% 감소했습니다.
*   API 할당량을 영상 추적이 아닌, 더 넓은 범위의 트렌드 발굴(300개 스냅샷)에 집중적으로 배분할 수 있게 되었습니다.

### 3. 알림 데이터의 단순화
*   복잡한 Z-Score 계산을 배제하고, 직관적인 **Power(성장)**와 **Ratio(공급)** 지표를 기준으로 즉각적인 알림(SURGE, POWER)을 생성합니다.

---
**설계 가이드**: 새로운 트렌드 지표가 필요할 경우 `trend_analysis` 테이블에 컬럼을 추가하거나 분석 함수를 보정하는 방식으로 확장이 가능합니다.
